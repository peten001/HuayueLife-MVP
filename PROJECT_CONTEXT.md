# 华越优选 HuayueLife-MVP

## 项目定位

华越优选是服务越南北宁、北江华人用户和华人餐厅的轻量餐饮点餐平台。

核心场景：

- 浏览附近餐厅
- 扫描桌码点餐
- 堂食
- 到店自取
- 商家配送
- 查看和跟踪订单

项目不是综合本地生活平台，也不是后台管理工具或单一扫码工具。V1 不包含在线支付、骑手、优惠券、积分和其他行业模块。

## 当前技术架构

- Monorepo：pnpm workspace
- MiniApp：UniApp、Vue 3、TypeScript、Pinia，首发微信小程序
- 商家后台：Vue 3、Vite、TypeScript、Pinia
- API：NestJS、Prisma
- 数据库：MySQL 8
- 共享模块：`packages/shared`
- 金额：越南盾整数 `VND`，禁止浮点金额
- 时间：数据库保存 UTC，界面按 `Asia/Ho_Chi_Minh` 展示

## MiniApp 产品目标

- 让用户快速发现附近可用餐厅。
- 让堂食用户扫码后明确知道当前商家和桌号。
- 让用户以尽量少的步骤完成选菜、购物车和下单。
- 让商家、菜品、价格、配送方式和订单状态清晰可信。
- 形成成熟餐饮产品感，同时保持微信小程序的轻量和熟悉感。

## MiniApp UI 方向

整体关键词：

- 清爽
- 年轻
- 餐饮感
- 健康
- 舒服
- 微信小程序风格

设计接近轻量版美团或饿了么的信息效率，但不采用红色平台风格。页面不能呈现后台管理系统感，也不能只像一个扫码工具。

核心表现：

- 卡片化布局
- 大幅、真实的餐饮和商家图片
- 少而有效的文字
- 明确的价格、营业状态、配送方式和桌号
- 高识别度的主要操作按钮
- `12-20px` 圆角，默认建议 `16px`
- 统一、克制的轻量阴影
- 通过留白和背景层级分组，减少边框使用

## 颜色系统

- 主色：`#43A047`
- 辅助深绿：`#2E7D32`
- 浅绿色背景：`#EAF7EE`
- 页面背景：`#F6FAF7` 或 `#F8FBF8`
- 白色卡片：`#FFFFFF`
- 橙黄色点缀：`#FFB74D`
- 主文字：`#1F2D24`
- 次级文字：`#666666`
- 浅灰边框：`#EEEEEE` 或 `#F0F0F0`

颜色使用原则：

- 主色用于品牌、主要按钮、选中状态和关键入口。
- 深绿用于强调文字、按压态或需要更强对比的绿色区域，避免大面积铺满。
- 浅绿用于轻提示、选中背景和局部品牌氛围，避免医院感。
- 橙黄色用于评分、热销、轻促销和餐饮氛围点缀，不与主绿色争夺层级。
- 红色仅用于错误、危险、取消等语义状态。

明确禁止：

- 大面积红色或美团红视觉方向
- 将 `#C62828` 定义为品牌主色
- 医院感浅绿
- 大面积深墨绿
- 荧光绿

## 布局与组件规范

- 移动端优先，以常见微信小程序设备宽度设计。
- 使用 8pt Grid，主要间距采用 `8/16/24/32px`，微间距可使用 `4px`。
- 页面水平边距通常为 `16px`。
- 主要按钮高度不低于 `48px`，点击区域不小于 `44px × 44px`。
- 商品和商家图片使用稳定比例、统一圆角和 `aspectFill` 类裁切策略。
- 卡片以白色为主，边框仅在确有分隔需求时使用浅灰色。
- 搜索、分类、购物车、结算和扫码入口必须有清晰的可操作状态。
- 页面需定义加载、空数据、错误、禁用、售罄和长文本表现。
- 底部购物车和结算操作区必须适配安全区域。

## 页面设计流程

开发或重构 MiniApp 页面前，先输出：

1. 页面目标
2. 用户场景
3. 页面结构
4. 设计理由

然后再编写代码。页面实现应优先保证用户体验、视觉一致性、商业产品感和餐饮行业习惯。

## 技术边界

- 保持 UniApp、Vue 3 和 TypeScript。
- 不引入 React、Next.js 或 Shadcn React 组件。
- 可参考 Shadcn 的留白、状态、组件组合和一致性，但必须转换为 UniApp 实现。
- 使用微信小程序支持的 API 和样式能力，不假设浏览器 DOM 或 H5 专属 CSS 可用。
- 当前页面中存在的暖红色样式属于历史实现；后续只有在明确的 UI 改造任务中才能迁移。本次上下文修正不修改页面代码。
